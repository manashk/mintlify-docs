## Overview

Vaani uses environment variables for all configuration. Variables are loaded via `pydantic-settings` with optional `.env` file support.

**`.env` file loading:** Automatic in `local`, `dev`, and `development` environments. Disabled in production (use Docker `env_file` or system env vars).

## Security Notes

<Warning>
- All `SecretStr` fields are masked in logs and debug output
- Never commit `.env` files to version control
- In production, use Docker `env_file` directive or platform-native secrets management
- The `JWT_API_SECRET` **must be identical** across backend and UI
</Warning>
